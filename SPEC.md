**X-02A ワイバーン / CFA-44 ノスフェラトゥ / ADFX-01 モルガン / XFA-27**

- **第4世代機**として追加（year=2035相当）
- **全国共通機**（特定の国でなく、研究で誰でも解放）
- 役割分担は**エスコン原作設定に準拠**
- 架空兵装は**専用兵装が明確な機体だけ新規モジュールを作る**（CFA-44・ADFX-01）
- 3Dモデルは**共通の戦術機モデル**を使う（全機共通＝個別流用は不要と判明）
- 画像はユーザー提供（C:\Users\tkmuh\Downloads\ACES）

## 各機の設計

### ① X-02A ワイバーン（制空ステルス型）
- **役割**：高機動・ステルス・万能制空。空戦特化。
- **性能傾向**：機動↑・対空↑・装甲は中（ADF-01比でspeed/air_attack寄り）
- **装備（13スロット・既存武装の組み合わせ）**：
  - 装甲=複合装甲 / OS=第4世代 / エンジン=第4世代 / 頭部=空戦型
  - 右手=突撃砲3 / 左手=反応装甲シールド
  - 頭部追加=高性能センサー3（ステルス索敵）/ 肩=スラスター3
  - 左背中=長刀 / 右背中=長刀 / 腕=チェーンソード / 腰=スラスター3 / 脚=追加装甲
- **架空兵装**：なし（既存組み合わせで表現）

### ② CFA-44 ノスフェラトゥ（多弾頭火力型）★架空兵装あり
- **役割**：大量ミサイルによる広域火力。
- **性能傾向**：対空・対地火力↑・コスト↑（重武装）
- **架空兵装（新規）**：**ADMM（全方位多目標ミサイル）**
  - カテゴリ=`add_missile_type`（肩スロット装備）
  - 既存 large_missile_2 を上回る多弾頭。対空・対地ともに高威力
- **装備**：肩=ADMM / 右手=突撃砲3 / 左背中=追加突撃砲3 / 左手=反応装甲シールド / 他=センサー・スラスター

### ③ ADFX-01 モルガン（実験レーザー型）★架空兵装あり
- **役割**：実験的なレーザー兵装機。
- **架空兵装（新規）**：**TLS（戦術レーザーシステム）**
  - F-47の荷電粒子砲を参考に、専用カテゴリ`ESC_TLS_type`で作成（右手の代替大型兵装）
  - 高い貫通（ap_attack）・対空。原作のTLS／ZOEをイメージ
  - 必要なら専用スロットでなく既存の右手スロットに収める形にして素体改変を避ける
- **装備**：右手=TLS（新規）/ 頭部追加=高性能センサー / 肩=スラスター / 装甲厚め

### ④ XFA-27（高機動汎用型）
- **役割**：バランス重視・近接戦も可能な汎用試作機。
- **性能傾向**：ADF-01同等のバランス型
- **装備**：右手=突撃砲3 / 左手=長刀 / 腕=チェーンソード / 肩=スラスター / バランス配分
- **架空兵装**：なし（既存組み合わせ）

## 実装手順
1. 画像加工：4機のJPEGから上半身を切り出し→228×104のDDS化（CFA-44は左1体）
2. spriteType定義：`GFX_MVLV_pic_<機体ID>` を interface に追加
3. 新規モジュール定義：ADMM・TLS（必要ならカテゴリ追加）
4. 機体equipment定義：4機を chassis_4 ベースで追加（性能・default_modules）
5. 解放テク：`tsf_common_<機体>` 隠しテックに enable_equipments（+架空兵装は enable_equipment_modules）
6. on_actions：tsf_4_framework研究で全国付与（既存共通機パターンに追加）
7. localisation：機体名（例「X-02A ワイバーン」）
8. 検証：括弧・BOM・スロット適合・カテゴリ適合・picture解決・解放整合
9. デバッグ：新規ゲームでresearch all→落ちない＋4機が解放・生産・配備できるか

## 注意・既知のリスク（KNOWLEDGE.md準拠）
- common/*.txt はBOM無し、loc ymlはBOM有
- 新規カテゴリを作る場合、そのカテゴリを許可するスロットが素体に必要（無いとデザイナーで装備不可）→ TLSは既存right_handに寄せるのが安全
- default_modulesのスロットは素体の実在スロットのみ、カテゴリ適合必須
- 装備IDにハイフン不可（X-02A→内部IDは `X02A` 等）
- 解放テクに機体を二重登録しない
